SEO

Google is using page speed in their ranking algorithm for SEO, how can we make our websites faster?

Method #1 - Optimize images

Images must have the appropriate resolution depending on the size of the image shown on the website.

Max resolution

Size: 17,9 MB

Format: PNG

This uncompressed 4K image in PNG format overloads the web page, is a bad practice; consequently, you are losing SEO. PNG format is used when transparencies are required (e.g., logos).

Compressed

Size: 27,1 KB

Format: JPG

The same image, but with 400px resolution on the horizontal axis, the image must be optimized to save the maximum memory and improve the loading speed, JPG format weighs much less than PNG.

Ultra Compressed

Size: 7,2 KB

Format: JPG

Optimize, yes, but not too much. In this image you can see that it has been compressed too much and has lost image quality, so it would not be suitable. You have to optimize between image quality and weight.

Method #2 - Minify files

To compress files (.html, .css, .js) you can use tools to minimize them automatically (Grunt, Gulp, among others).

Method #3 - Static content with CDNs

Because the location of the user impacts page load speed, spreading your content across servers will speed up this process.

HOISTING

What is hoisting?

Hoisting is one of the peculiarities of JavaScript, and is given when we define a variable inside a function (in the scope), then the internal interpreter declares it at the beginning of its context.

Important factor: JavaScript only hoists declarations, not initializations.

Depending on the scope, the value of the variable will change. When "f(1)" is executed, the variable updated "color", in fact, is the one that is inside the "if"; even if the condition is false, the declaration variable inside will hoist outside the conditional statement.

In the first example, hoisting acts normally, the variable "f" (function) is declared, but it has no value when called.

In the second case, unlike variables, a function declaration doesn't just hoist the function's name. It also hoists the actual function definition.

Tip: variables has to be declared on the top of the script, but functions can be declared on the bottom.
